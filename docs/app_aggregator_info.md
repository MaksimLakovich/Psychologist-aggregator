# 🛠️ Агрегатор системы (фильтрация и ранжирование данных в публичном каталоге психологов)

Задача aggregator: показывать профиль наружу - красиво, фильтровать, искать, ранжировать.

---

## <a id="title1"> 🔑 Основные особенности </a>

- Первоначальный подбор психологов (пользовательская фильтрация по: topics, methods, age, gender, price range,
  experience range)
- Итоговая фильтрация на основе предпочитаемых клиентом временных слотов
- Ранжирование (алгоритм выдачи) - рандомный вывод карточек психологов в "Публичном каталоге психологов" (
  is_verified=True, is_active=True)
- Дополнительная scoring-модель (relevance алгоритм):
    - повышение позиции в выводе по "last_login" - т.е. повышение положения психологов, которые чаще работают с
      приложением
    - повышение позиции в выводе по "rating" + "уровень совпадений интересующих тем клиента в requested_topics и тем
      психолога в "topics" + аналогичное совпадение в методах - "preferred_methods" и "methods")
- Рекомендации системы (продвижение) / продвинутый boost (подписка)

```text
ClientProfile
      ↓
aggregator.MatchPsychologistsAjaxView
      ↓
aggregator.PsychologistAggregatorService
 - если has_match == False → психолог исключается
 - если has_match == True → психолог остается + availability передается в контекст
      ↓
aggregator.basic_filter_service.py    (Первичная фильтрация по: topics, methods, age, gender, price range, experience range)
      ↓
QuerySet[PsychologistProfile]    (ORM / SQL)
      ↓
calendar_engine.generate_and_match_factory    (Собирает use-case генерации и matching доступности для одного психолога)
      ↓
calendar_engine.GenerateAndMatchAvailabilityUseCase    (фильтрация по: selected_slots)
      ↓
MatchResultDTO    (Python-объект)
      ↓
aggregator.scoring.py    (Финальное ранжирование по коэфф совпадения тем/методов среди отсортированных психологов)
      ↓
JSON

```

## <a id="title3"> 🖥️ WEB-функционал </a>

### 1. ФИЛЬТРЫ

### aggregator/_web/selectors/:

Для **WEB-эндпоинтов** созданы следующие методы для фильтрации:

| № | Модуль                    | Метод                         | Описание функционала (docstring)                                                                                                                                                                                                                 |
|---|---------------------------|-------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1 | psychologist_selectors.py | `base_queryset()`             | Метод возвращает базовый QuerySet, содержащий ВСЕХ активных/верифицированных психологов из базы данных                                                                                                                                           |
| 2 | psychologist_selectors.py | `filter_by_topic_type()`      | Метод оставляет психологов, у которых в профиле есть хотя бы одна тема (topics) из заданного клиентом *Вида консультации* (individual/couple). Это нужно для того, если клиент указал вид консультации, но не выбрал ни одной темы из этого вида |
| 3 | psychologist_selectors.py | `annotate_topic_matches()`    | Метод аннотирует поле matched_topics_count, где подсчитывает количество совпадающих тем из профиля психолога (topics) с темами из профиля клиента (requested_topics)                                                                             |
| 4 | psychologist_selectors.py | `annotate_type_topic_count()` | Метод аннотирует поле type_topics_count. Данные о том сколько у каждого психолога указано в его профиле тем из указанного клиентом *Вида консультации*: individual / couple (полезно когда requested_topics пуст)                                |
| 5 | psychologist_selectors.py | `filter_by_gender()`          | Метод с жесткой фильтрацией по полу психолога (мужчина/женщина)                                                                                                                                                                                  |
| 6 | psychologist_selectors.py | `filter_by_age()`             | Метод с жесткой фильтрацией по возрасту психолога                                                                                                                                                                                                |
| 7 | psychologist_selectors.py | `annotate_method_matches()`   | Метод аннотирует поле matched_methods_count, где подсчитывает количество совпадающих методов из профиля психолога (methods) с методами из профиля клиента (preferred_methods)                                                                    |

### 2. СЕРВИСЫ (СКОРИНГ + ИТОГОВАЯ ОБЩАЯ ФИЛЬТРАЦИЯ С РАНЖИРОВАНИЕМ)

### aggregator/_web/services/:

| № | Модуль                 | Метод                      | Описание функционала (docstring)                                                                                                                                                                                                                                                                                          |
|---|------------------------|----------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1 | scoring.py             | `topic_score()`            | Метод аннотирует поле topic_score (коэффициент совпадения тем)                                                                                                                                                                                                                                                            |
| 2 | scoring.py             | `method_score()`           | Метод аннотирует поле method_score (коэффициент совпадения методов)                                                                                                                                                                                                                                                       |
| 3 | scoring.py             | `apply_final_ordering()`   | Метод итоговой сортировки психологов. Логика: 1) topic_score - главный критерий; 2) method_score - вторичный критерий                                                                                                                                                                                                     |
| 4 | basic_filter_service.py | `match_psychologists()`    | Первичная фильтрация. Метод возвращает итоговый QuerySet, содержащий психологов отсортированных по коэффициенту совпадения тем, полу, возрасту и коэффициенту совпадения методов                                                                                                                                          |
| 5 | final_aggregator.py    | `PsychologistAggregatorService` | Центральный сервис агрегации: <br/> - запуск первичной фильтрации match_psychologists(); <br/> - финальная фильтрация по selected_slots от пользователя и AvailabilityRule от специалиста; <br/> - финальное ранжирование по коэффициенту совпадения тем и методов (scoring.py); <br/> - подготовка данных для API / AJAX |
