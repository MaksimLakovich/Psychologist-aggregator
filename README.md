## <a id="title1"> 📌 О проекте </a>
***Aggregator for finding a psychologist*** - это web-приложение для поиска, подбора и записи к психологу.


### 1. Пользовательские роли:

| Роль              | Основные права                                                                                                                                                                     | Дополнительные права                                                               |
| ----------------- |------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------|
| **Клиент**        | Регистрация/вход, редактирование профиля, поиск психологов, запись на сеанс, оплата, просмотр истории сеансов, оставление отзыва                                                   | Подписка (пакет сеансов), отмена/перенос записи, получение уведомлений             |
| **Психолог**      | Регистрация, заполнение профиля, настройка расписания, подтверждение/отмена записи, прием клиентов (видео-комната), получение оплаты, просмотр отзывов                             | Управление доступностью календаря, управление ценами сеанса, получение уведомлений |
| **Администратор** | Управление пользователями, блокировка/активация аккаунтов, модерация новых психологов на платформе, контроль оплат/подписок, работа со статистикой и рейтингами, модерация отзывов | Настройка тарифов и комиссий платформы                                             |


### 2. Функциональные требования:

| Функциональность                  | Описание                                                                                                              |
|-----------------------------------|-----------------------------------------------------------------------------------------------------------------------|
| **Регистрация / Авторизация**     | JWT-токены, подтверждение email/phone, роли пользователей, throttle (anti-spam) для отправки email                    |
| **Профиль пользователя**          | Общие данные (просмотр и редактирование данных)                                                                       |
| **Карточка психолога**            | Просмотр и редактирование данных, фото, стаж, образование, компетенции (для психологов)                               |
| **Поиск и фильтрация психологов** | По критериям (специализация, опыт, стоимость, пол, онлайн/оффлайн, расписание и т.д.)                                 |
| **Календарь и записи**            | Психолог создает расписание, клиент бронирует слот (создать/перенести/отменить)                                       |
| **Соглашения / ПДн**              | Сбор и хранение (шифрование): Согласие/ ПДн / Пользовательское соглашение / Договор оказания услуг                    |
| **Оплата / Подписки**             | Внести предоплату/полная оплата - пока фиктивная (эмуляция оплаты) с последующей заменой на реальный платежный шлюз   |
| **Отзывы / Оценки**               | После сеанса клиент оставляет отзыв                                                                                   |
| **Уведомления / Напоминания**     | Уведомления о записях, изменениях, отменах (email + push/SMS + Telegram-bot)                                          |
| **Админ-панель**                  | Управление пользователями, статистика по оплатам, информация о подписках, базовая аналитика, управление справочниками |
| **Аналитика / Отчеты**            | Просмотр активности, рейтингов, выручки (админ)                                                                       |


### 3. Нефункциональные требования:

| Категория                 | Требование                                                                                                                                                                                                                                                                                         |
| ------------------------- |----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Производительность**    | Время отклика API ≤ 200 мс для основных запросов                                                                                                                                                                                                                                                   |
| **Безопасность**          | Хранение паролей (PBKDF2 / Argon2), HTTPS, CSRF, CORS, rate limiting<br/>- TLS 1.2/1.3, HSTS, регулярно обновляемые сертификаты.<br/>- Защита от CSRF/XSS/SQLi - стандартные практики Django + проверка входящих данных. <br/>- Rate limiting (throttling) на эндпоинты аутентификации и на поиск. |
| **Хранение данных**       | PostgreSQL (в Yandex Cloud) + отдельный storage для FileField, ImageField (в Yandex Object Storage), Redis для кэша/очередей                                                                                                                                                                       |
| **Резервное копирование** | Бэкап PostgreSQL ежедневно                                                                                                                                                                                                                                                                         |
| **Соблюдение ФЗ-152**     | Согласие пользователя на обработку ПДн, данные хранятся в РФ                                                                                                                                                                                                                                       |
| **Масштабирование**       | Возможность развертывания с Docker Compose, далее - K8s                                                                                                                                                                                                                                            |
| **Мониторинг**            | Логирование в файл + Yandex Monitoring (или self-hosted Grafana)                                                                                                                                                                                                                                   |


### 4. Аналитика и админка:
- Метрики использования: конверсии поиска -> бронирование, средний чек, retention, NPS. 
- Админ-панель для модераторов (отслеживания платежей/списков/модерации профилей). 
- Экспорт отчетов (CSV/XLSX) для бухгалтерии/маркетинга.

## <a id="title2"> ⚙️ Технологии </a>

### Архитектура проекта:

| Компонент          | Инструмент                                                                                                     | Комментарий                                                                                                                          |
|--------------------|----------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------|
| **Backend**        | **Python 3.11+, Django 5.x, Django REST Framework**                                                            | Основной API, модели, бизнес-логика                                                                                                  |
| **БД**             | **PostgreSQL (Yandex Managed PostgreSQL + отдельный storage для FileField/ImageField в Yandex Object Storage)** | Основное хранилище данных                                                                                                            |
| **Кеш / брокер**   | **Redis (через django-redis)**                                                                                 | Кеш, хранение сессий, Celery broker                                                                                                  |
| **Фоновые задачи** | **Celery**                                                                                                     | Рассылки, уведомления, напоминания                                                                                                   |
| **WSGI-сервер**    | **Gunicorn + Nginx**                                                                                           | Продакшн-развертывание (сервер и статика)                                                                                            |
| **Контейнеризация** | **Docker + docker-compose**                                                                                    | Сборка и локальный деплой                                                                                                            |
| **Хостинг**        | **Yandex.Cloud (Compute VM + Object Storage)**                                                                 | Российская инфраструктура                                                                                                            |
| **CI/CD**          | **GitHub Actions (позже - self-hosted Runner в Yandex.Cloud)**                                                 | Автоматизация сборки и деплоя                                                                                                        |
| **Email / SMS**    | **SMTP (Яндекс 360) / на первом этапе mock-SMS (вопрос бюджета)**                                              | Подтверждения регистрации, уведомления                                                                                               |
| **SSR (рендеринг)** | Django Templates                                                                                               | Страницы генерируются сервером (Server-Side Rendering)                                                                               |
| **CSS/HTML**       | **Tailwind + Flowbite + DaisyUI + HyperUI**                                                                    | MVP UI без необходимости SPA - через Django templates <br/> **SSR** - это Server-side rendering (рендеринг страниц на стороне Django) |
| **JavaScript**     | Vanilla JS + Fetch API + HTMX (постепенно) + FullCalendar.js                                                   | Интерактив: autosave чего-то в БД, AJAX, локальная логика (например, свернуть/развернуть)                                            |
| **Frontend**       | (этап 2)                                                                                                       | Развитие UI (2 этап): разработка SPA/MPA-приложения на чем-то из: React, Vue или Next.js и т.д.                                      |


### Дополнительные пояснения:
1. `API-решение:`
   - **Django REST Framework**
2. `WEB-решение:`
   - **Django templates** - основа
   - **Tailwind** - современный UI (система стилей)
   - **Flowbite** - сложные продвинутые компоненты (модалки, таблицы, datepicker, navbar)
   - **DaisyUI** - UI-компоненты (кнопки, поля, карточки) 
   - **HTMX** - динамика/интерактив без React (календарь, фильтры, списки, пагинация, сортировка)
   - **FullCalendar.js** - JS-календарь (есть drag&drop, поддержка событий, блокировка занятых слотов)

### 2.1. Приложение `users`:

- ℹ️ **Детальная информация о приложении *"Users"*: [Users (app_users_info.md)](docs/app_users_info.md).**
- ⚠ **Описание значений в справочнике `Topic`: [fixture_topics_info.md](docs/fixture_topics_info.md).**
- ⚠ **Описание значений в справочнике `Method`: [fixture_methods_info.md](docs/fixture_methods_info.md).**
- ⚠ **Описание значений в справочнике `Specialisation`: [fixture_specialisations_info.md](docs/fixture_specialisations_info.md).**


- Список сущностей (моделей):
  - `TimeStampedModel`: это абстрактная базовая модель для дальнейшего создания *created_at* и *updated_at* во всех моделях приложения при необходимости.
  - `UserRole`: представляет роли пользователей в приложении.
  - `Topic`: представляет тему/вопрос (запрос на терапию) - то с чем клиент приходит.
  - `Specialisation`: представляет специализацию (методологическая школа) - то в чем психолог специализируется.
  - `Method`: представляет метод (инструмент/подход), который использует психолог в работе.
  - `Education`: представляет образование, которое есть у психолога.
  - `AppUser`: представляет пользователя приложения.
  - `PsychologistProfile`: расширяется от AppUser и представляет профиль психолога.
  - `ClientProfile`: расширяется от AppUser и представляет профиль клиента.


- Структура связей:
    ```python
    AppUser 1───1 PsychologistProfile
    AppUser 1───1 ClientProfile
    AppUser *───1 UserRole
    PsychologistProfile *───* Specialisation
    PsychologistProfile *───* Method
    PsychologistProfile *───* Topic
    PsychologistProfile *───* Education
    ClientProfile *───* Topic
    ClientProfile *───* Method
    ```


### 2.2. Приложение `calendar_engine`:

- ℹ️ **Детальная информация о приложении *"Calendar_engine"*: [Calendar (app_calendar_info.md)](docs/app_calendar_info.md).**


- Список сущностей (моделей):
  - `TimeStampedModel`: это абстрактная базовая модель для дальнейшего создания *created_at* и *updated_at* во всех моделях приложения при необходимости
  - `CalendarEvent`: событие календаря
  - `RecurrenceRule`: правила повторений
  - `TimeSlot`: временной слот события
  - `EventParticipant`: участник события
  - `SlotParticipant`: участник конкретного слота (если отличается от события)
  - `AvailabilityRule`: правила доступности: рабочее расписание
  - `AvailabilityRuleTimeWindow`: временное окно доступности внутри рабочего дня из AvailabilityRule
  - `AvailabilityException`: исключения из расписания
  - `AvailabilityExceptionTimeWindow`: переопределенное временное окно доступности внутри рабочего дня из AvailabilityException

- Структура связей:
    ```python
    AppUser 1───* CalendarEvent                              # пользователь может организовать много событий
    CalendarEvent 1───* TimeSlot                             # событие состоит из 1+ слотов
    CalendarEvent 1───* RecurrenceRule                       # у события может быть несколько правил повторения для разных событий
    
    AppUser (через EventParticipant) *───* CalendarEvent     # участие пользователей в событиях
        # Пояснение:
            # поддержка групповых событий
            # роли и статусы находятся в EventParticipant
            # один пользователь может участвовать в тысячах событий
    AppUser (через SlotParticipant) *───* TimeSlot           # участие пользователей в конкретных слотах
        # Пояснение:
            # используется только при необходимости
            # позволяет разный состав участников по слотам
            # критично для курсов, переносов, аналитики
  
    AppUser 1───1 AvailabilityRule
    AvailabilityRule 1───* AvailabilityRuleTimeWindow
    AvailabilityRule 1───* AvailabilityException
    AvailabilityException 1───* AvailabilityExceptionTimeWindow
    AppUser 1───* AvailabilityException
        # Пояснение:
            # AvailabilityRule: базовое рабочее расписание 
            # AvailabilityRuleTimeWindow: доступные рабочие окна внутри AvailabilityRule (например, с 07 до 11 и потом с 16 до 22)
            # AvailabilityException: привязано к рабочему расписанию; может быть глобальным (отпуск, больничный)
            # AvailabilityExceptionTimeWindow: переопределенные рабочие окна внутри AvailabilityException
            # availability никогда не равно бронированию
    ```
